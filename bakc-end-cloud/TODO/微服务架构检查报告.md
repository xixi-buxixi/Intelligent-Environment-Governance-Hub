## 一、严重问题（必须修复）

### 1.1 服务名不一致问题

**问题描述**：各服务的 `bootstrap.yaml` 和 `application.yaml` 中服务名不一致，会导致服务注册发现失败。

| 服务 | bootstrap.yaml | application.yaml | Gateway路由 | 状态 |
|------|----------------|------------------|-------------|------|
| user-service | `userService` | `userService` | `lb://userService` | ⚠️ 部分正确 |
| assistant-service | `assistantService` | `assistant-service` | `lb://assistant-service` | ❌ 不一致 |
| data-service | `dataService` | `data-service` | `lb://data-service` | ❌ 不一致 |

**影响**：
- Nacos服务注册名与Gateway路由目标名不匹配
- 服务间Feign调用找不到目标服务
- 请求路由失败

**解决方案**：

统一使用 kebab-case 命名（推荐）：

```yaml
# user-service/bootstrap.yaml
spring:
  application:
    name: user-service  # 统一改为 user-service

# assistant-service/bootstrap.yaml
spring:
  application:
    name: assistant-service  # 统一改为 assistant-service

# data-service/bootstrap.yaml
spring:
  application:
    name: data-service  # 统一改为 data-service
```


### 1.4 缺少 Feign 客户端定义

**问题描述**：服务间调用没有定义 Feign 客户端接口。

**原始项目的服务调用关系**：

```
DataFetchController -> UserService (获取用户信息)
```

**解决方案**：

如果 data-service 需要调用 user-service 获取用户信息，需要创建 Feign 客户端：

```java
// data-service/src/main/java/com/environment/dataservice/client/UserServiceClient.java
package com.environment.dataservice.client;

import com.environment.dataservice.fallback.UserServiceFallback;
import org.springframework.cloud.openfeign.FeignClient;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestHeader;

@FeignClient(value = "user-service", fallback = UserServiceFallback.class)
public interface UserServiceClient {

    @GetMapping("/user/info")
    UserInfo getCurrentUser(@RequestHeader("Authorization") String token);
}
```

### 2.3 认证逻辑需要重构

**原始项目的问题**：

`DataFetchController.java` 中直接解析 JWT：

```java
private User getCurrentUser(HttpServletRequest request) {
    String token = request.getHeader("Authorization");
    // ... 直接解析 JWT
}
```

**微服务架构建议**：

1. **Gateway 层**：解析 JWT，提取用户信息放入 Request Header
2. **业务服务**：从 Header 中获取用户信息

Gateway 的 `AuthGlobalFilter.java` 应该：

```java
@Component
public class AuthGlobalFilter implements GlobalFilter, Ordered {

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        String token = exchange.getRequest().getHeaders().getFirst("Authorization");

        if (token != null && token.startsWith("Bearer ")) {
            Claims claims = JwtUtil.parseToken(token.substring(7));
            if (claims != null) {
                // 将用户信息放入 Header 传递给下游服务
                ServerHttpRequest request = exchange.getRequest().mutate()
                    .header("X-User-Id", claims.getSubject())
                    .header("X-User-Name", claims.get("username", String.class))
                    .header("X-User-Role", claims.get("role", String.class))
                    .build();
                return chain.filter(exchange.mutate().request(request).build());
            }
        }

        return chain.filter(exchange);
    }
}
```

## 三、Nacos 配置检查

您提供的 Nacos 配置：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/${spring.application.name}?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root
    password: 200575
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848
    sentinel:
      transport:
        dashboard: localhost:8858
        port: 8719
      eager: true
feign:
  sentinel:
    enabled: true
mybatis-plus:
  mapper-locations: classpath*:/mapper/**/*.xml
  type-aliases-package: com.environment.*.pojo
  configuration:
    map-underscore-to-camel-case: true
  global-config:
    db-config:
      id-type: auto
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
```

### 配置问题分析

| 配置项 | 问题 | 解决方案 |
|--------|------|----------|
| `${spring.application.name}` | 服务名不一致会导致数据库名混乱 | 统一服务名后再使用 |
| `password: 200575` | 明文密码，安全风险 | 使用 Nacos 加密配置或环境变量 |
| 缺少 Redis 配置 | assistant-service 需要 Redis | 添加 Redis 配置 |
| 缺少 LangChain4j 配置 | assistant-service 需要 AI 配置 | 添加 LangChain4j 配置 |

### 建议的 Nacos 公共配置

```yaml

    password: ${DB_PASSWORD:200575}  # 使用环境变量覆盖
  data:
```

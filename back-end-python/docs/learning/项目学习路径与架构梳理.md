## 二、各阶段详细流程

### 阶段一：RAG基础（查知识）

**核心问题**：如何让AI能查询本地知识库？

```
流程图：

用户提问 ──▶ 检索器(Retriever) ──▶ 从向量库找相关文档 ──▶ 
                                                    │
                                                    ▼
                         LLM生成回答 ◀── 拼接上下文(Context) ◀── 文档内容

涉及文件：
- src/config.py          → 配置API密钥
- src/document_processor.py → 加载和分割文档
- src/vectorstore.py     → 向量数据库管理
- src/rag_chain.py       → RAG问答链
```

**关键类与调用关系**：

```
【调用入口】main.py 或 app.py

step1: 创建文档处理器
       processor = DocumentProcessor(chunk_size=500, chunk_overlap=50)

step2: 加载文档并分割
       chunks = processor.process("./data/knowledge")

step3: 创建向量存储
       manager = VectorStoreManager(persist_directory="./chroma_db", api_key="...")
       vectorstore = manager.create_from_documents(chunks)

step4: 获取检索器
       retriever = manager.get_retriever(top_k=3)

step5: 创建RAG链（注入检索器）
       rag_chain = RAGChain(retriever=retriever, model_name="qwen-plus", api_key="...")

step6: 执行问答
       answer = rag_chain.invoke("什么是垃圾分类？")
```

**关键注入点**：`retriever` 被注入到 `RAGChain` 的构造函数中。

```
def run_tool_call_loop(llm_with_tools, tools_list, question):
    messages = [HumanMessage(content=question)]
    
    while True:
        ai = llm_with_tools.invoke(messages)
        
        if not ai.tool_calls:  # 无工具调用，直接回答
            return ai.content
        
        # 有工具调用，执行并回填
        messages.append(ai)
        for call in ai.tool_calls:
            tool = tools_by_name[call["name"]]
            result = tool.invoke(call["args"])
            messages.append(ToolMessage(content=str(result), tool_call_id=call["id"]))
```

## 三、核心问题解答：类如何注入到Agent？

### 问题：每次写好了需要调用的class类，是在哪进行调用的？

**答案**：主要通过以下三种方式注入：

---

### 方式一：构造函数注入（最常见）

```
┌────────────────────────────────────────────────────────────────────┐
│  构造函数注入示例                                                    │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【定义类】                                                          │
│  class RAGChain:                                                    │
│      def __init__(self, retriever, model_name, api_key):            │
│          self.retriever = retriever  ← 注入检索器                   │
│          self.llm = ChatTongyi(...)                                 │
│          self.chain = self._build_chain()                           │
│                                                                     │
│  【调用类】                                                          │
│  retriever = manager.get_retriever(top_k=3)                         │
│  rag_chain = RAGChain(                                              │
│      retriever=retriever,      ← 传入检索器实例                     │
│      model_name="qwen-plus",                                        │
│      api_key=Config.OPENAI_API_KEY                                  │
│  )                                                                  │
│                                                                     │
│  【注入的是什么？】                                                   │
│  retriever 是一个"能力"，RAGChain通过它获得"检索知识"的能力          │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

**实际代码位置**：

```python
【文件: src/rag_chain.py:702-730】

class RAGChain:
    def __init__(
        self,
        retriever,              ← 检索器被注入这里
        model_name: str = "qwen-plus",
        temperature: float = 0.7,
        api_key: str = None,
    ):
        self.retriever = retriever  ← 保存注入的检索器
        
        # 创建LLM
        self.llm = ChatTongyi(
            model_name=model_name,
            temperature=temperature,
            dashscope_api_key=api_key,
        )
        
        # 使用注入的retriever构建链
        self.chain = self._build_chain()
    
    def _build_chain(self):
        chain = (
            {
                "context": self.retriever | format_docs,  ← 使用注入的检索器
                "question": RunnablePassthrough()
            }
            | self.prompt
            | self.llm
            | StrOutputParser()
        )
        return chain
```

```python
def create_environment_agent():
    """创建环境小助手Agent"""
    
    # 1) LLM配置
    llm = ChatTongyi(
        model_name="qwen-plus",
        dashscope_api_key=Config.OPENAI_API_KEY,
    )
    
    # 2) 工具列表注入
    tools = [
        safe_calculator,    ← 注入计算能力
        get_current_time,   ← 注入时间能力
        text_stats,         ← 注入统计能力

        model=llm,
        tools=tools,        ← 工具列表注入到这里
        system_prompt="你是环境小助手...",
    )
    
    return agent
```

## 七、关键理解点

### 1. "注入"的本质是什么？

**注入 = 给一个类传递另一个类的实例，让它获得新能力**

```python
# 不注入的情况
class Dog:
    def bark(self):
        return "汪汪"

# 注入的情况（Dog获得了"飞行"能力）
class Dog:
    def __init__(self, wings):  # wings被注入
        self.wings = wings
    
    def fly(self):
        return self.wings.flap()  # 使用注入的能力

# 实际使用
wings = Wings()
dog = Dog(wings=wings)  # 注入翅膀
dog.fly()  # 狗可以飞了！
```

### 2. Agent的"工具箱"是什么？

**工具箱 = 工具列表，Agent通过注入获得所有工具能力**

```python
# Agent好比一个工人
agent = create_agent(
    model=llm,           # 工人的大脑
    tools=[              # 工人的工具箱
        calculator,      # 计算器
        time_tool,       # 时钟
        rag_tool,        # 知识库
    ]
)

# 工人遇到问题时会选择合适的工具
# 用户问"1+2等于多少" → 工人拿出计算器 → 返回结果
```

### 3. 为什么RAG和Tools分开？

| RAG | Tools |
|-----|-------|
| **查知识**（被动检索） | **做事情**（主动执行） |
| 从本地知识库找答案 | 执行计算、查询时间等操作 |
| 返回文档内容 | 返回执行结果 |
| `retriever` 检索器 | `tool` 工具函数 |

**Agent可以同时拥有两者**：

```python
agent = create_agent(
    model=llm,
    tools=[
        rag_qa,          # RAG变成工具，Agent可以"查知识"
        calculator,      # 计算工具，Agent可以"做计算"
        time_tool,       # 时间工具，Agent可以"查时间"
    ]
)
```

## 八、常见困惑解答

### Q1: 我写了一个工具类，怎么让Agent用它？

**步骤**：

```python
# 1. 在 tools/ 下定义工具
@tool
def my_new_tool(param: str) -> dict:
    """工具描述"""
    return {"ok": True, "data": {...}}

# 2. 在 tools/__init__.py 导出
from .my_tools import my_new_tool
```

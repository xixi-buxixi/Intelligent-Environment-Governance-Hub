# 系统管理模块说明

## 模块概述

系统管理是智能环境治理中枢平台的核心功能模块，仅对 **ADMIN（管理员）** 角色开放。该模块提供用户管理、权限配置等功能，支持对系统用户进行增删改查及角色权限管理。

## 一、用户管理

### 1.1 功能说明

用户管理提供系统用户的全生命周期管理，包括：

| 功能 | 说明 |
|------|------|
| **用户列表** | 分页展示所有用户信息，支持按角色筛选 |
| **新增用户** | 创建新用户，设置基本信息和角色权限 |
| **编辑用户** | 修改用户信息（用户名不可修改） |
| **删除用户** | 逻辑删除用户，数据保留但标记为已删除 |

### 1.2 用户实体属性

```java
public class User {
    private Long id;           // 用户ID（主键）
    private String username;   // 用户名（唯一）
    private String password;   // 密码
    private String realName;   // 真实姓名
    private String email;      // 邮箱
    private String phone;      // 手机号
    private String department; // 部门
    private String role;       // 角色（ADMIN/USER/MONITOR）
    private Integer status;    // 状态（0-禁用，1-启用）
}
```

### 1.3 角色类型

| 角色代码 | 角色名称 | 说明 |
|----------|----------|------|
| `ADMIN` | 管理员 | 拥有系统所有权限，可管理用户和配置 |
| `USER` | 普通用户 | 拥有基本数据查看和提交权限 |
| `MONITOR` | 监测员 | 专注于环境监测数据的采集和查看 |

### 1.4 部门选项

- 信息技术部
- 环境监测部
- 生态保护部
- 污染防治部
- 运维部
- 管理部

## 二、权限配置

### 2.1 功能说明

权限配置模块用于查看和管理用户的角色权限：

- **权限查看**：以卡片形式展示各用户的角色和对应权限说明
- **权限修改**：管理员可通过密码验证修改用户角色

### 2.2 权限说明映射

| 角色 | 权限列表 |
|------|----------|
| **ADMIN** | 可进行用户管理、可进行权限配置、可查看所有数据、可进行系统设置 |
| **USER** | 可查看个人数据、可提交环保数据、可查看数据报表 |
| **MONITOR** | 可查看监测数据、可进行数据采集、可生成监测报告 |

### 2.3 权限修改安全机制

修改用户角色时需要进行**管理员密码验证**：

1. 选择目标用户
2. 选择新角色
3. 输入当前管理员密码进行身份确认
4. 验证通过后完成角色变更

## 三、接口设计

### 3.1 用户管理接口

| 接口路径 | 方法 | 功能 | 权限要求 |
|----------|------|------|----------|
| `/api/system/user/list` | GET | 获取用户列表（分页） | ADMIN |
| `/api/system/user/add` | POST | 新增用户 | ADMIN |
| `/api/system/user/update` | POST | 更新用户信息 | ADMIN |
| `/api/system/user/{id}` | DELETE | 删除用户 | ADMIN |
| `/api/system/user/{id}` | GET | 根据ID获取用户详情 | ADMIN |

### 3.2 权限配置接口

| 接口路径 | 方法 | 功能 | 权限要求 |
|----------|------|------|----------|

### 4.4 权限控制

页面加载时进行权限检查：

```javascript
const checkLogin = () => {
    const token = localStorage.getItem('token');
    const userInfo = localStorage.getItem('userInfo');

    if (!token || !userInfo) {
        // 未登录，跳转登录页
        window.location.href = 'login.html';
        return;
    }

    const user = JSON.parse(userInfo);

    // 非管理员禁止访问
    if (user.role !== 'ADMIN') {
        ElMessage.error('权限不足');
        window.location.href = 'home.html';
    }
};
```

## 五、后端实现

### 5.1 技术栈

- **框架**：Spring Boot 2.7
- **数据访问**：MyBatis
- **数据库**：MySQL 8.0
- **分页插件**：PageHelper
- **安全认证**：JWT

### 5.2 核心类说明

| 类 | 说明 |
|----|------|
| `UserController` | 用户管理API控制器 |
| `SystemController` | 系统管理API控制器 |
| `UserService` / `UserServiceImpl` | 用户业务逻辑层 |
| `UserMapper` | 用户数据访问层 |
| `RoleEnum` | 角色枚举定义 |
| `StatusEnum` | 状态枚举定义 |
| `PageDTO` | 分页请求参数 |
| `PageResult<T>` | 分页响应结果 |
| `Result<T>` | 统一响应结果 |

### 5.3 数据库表结构

```sql
CREATE TABLE `sys_user` (
    `id` BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '用户ID',
    `username` VARCHAR(50) NOT NULL COMMENT '用户名',
    `password` VARCHAR(100) NOT NULL COMMENT '密码',
    `real_name` VARCHAR(50) DEFAULT NULL COMMENT '真实姓名',
    `email` VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
    `phone` VARCHAR(20) DEFAULT NULL COMMENT '手机号',
    `department` VARCHAR(100) DEFAULT NULL COMMENT '部门',
    `role` VARCHAR(20) DEFAULT 'USER' COMMENT '角色',
    `status` TINYINT(1) DEFAULT 1 COMMENT '状态：0-禁用，1-启用',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    `deleted` TINYINT(1) DEFAULT 0 COMMENT '删除标志',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 5.4 安全设计

1. **JWT认证**：所有管理接口需要携带有效的JWT Token
2. **角色鉴权**：仅ADMIN角色可访问系统管理接口
3. **密码验证**：修改用户角色需验证管理员密码
4. **逻辑删除**：用户删除采用逻辑删除，保留数据完整性
5. **敏感信息过滤**：返回用户信息时清除密码字段

## 六、交互流程

### 6.1 用户登录访问系统管理

```
┌─────────┐    登录      ┌─────────┐   验证角色   ┌─────────┐
│  用户   │ ───────────> │  首页   │ ──────────> │系统管理 │
└─────────┘              └─────────┘             └─────────┘
                              │
                              │ 非ADMIN角色
                              ▼
                         ┌─────────┐
                         │ 拒绝访问 │
                         └─────────┘
```

### 6.2 用户管理操作流程

```
新增用户：点击新增 → 填写表单 → 验证 → 提交 → 刷新列表
编辑用户：点击编辑 → 加载数据 → 修改 → 提交 → 刷新列表
删除用户：点击删除 → 确认弹窗 → 提交 → 刷新列表
筛选查询：选择角色 → 点击搜索 → 刷新列表
```

### 6.3 权限修改流程

```
点击修改权限 → 选择新角色 → 输入管理员密码 → 验证 → 提交 → 刷新
```

## 八、注意事项

1. **权限控制**：系统管理模块严格限制为ADMIN角色访问，非管理员用户将被重定向到首页
2. **数据验证**：新增用户时用户名、密码、真实姓名、角色为必填项
3. **密码安全**：当前密码以明文存储，生产环境建议加密存储（MD5/BCrypt）
4. **删除机制**：用户删除为逻辑删除，数据库中数据保留，deleted字段标记为1
5. **会话管理**：Token有效期24小时，过期需重新登录

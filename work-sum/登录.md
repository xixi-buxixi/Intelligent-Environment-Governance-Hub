# 智能环境治理中枢平台 - 工作文档

## 项目概述

**项目名称**：智能环境治理中枢平台 (Intelligent Environment Governance Hub)

**创建时间**：2026年3月10日

**技术栈**：
- 前端：HTML + CSS + JavaScript + Vue.js 2.x
- 后端：Spring Boot 2.7 + MyBatis + MySQL
- 认证：JWT Token + 4位数字验证码

---

## 一、前端工作

### 1.1 项目结构

```
front-end/
├── login.html          # 登录页面
├── index.html          # 系统欢迎页
├── home.html           # 系统主页（功能模块入口）
├── css/
│   ├── login.css       # 登录页样式
│   └── home.css        # 主页样式
├── js/
│   ├── login.js        # 登录页逻辑（Vue）
│   └── home.js         # 主页逻辑（Vue）
└── images/
    └── login-bg.jpg    # 登录背景图片
```

### 1.2 登录页面 (login.html)

**功能特性**：
- 环保绿色主题设计，使用自定义背景图片
- 浮动粒子动画效果
- 用户名、密码输入框
- 4位数字验证码（点击刷新）
- 记住用户名功能
- 登录状态提示（成功/失败）
- 响应式布局，适配移动端

**技术实现**：
- Vue.js 2.x 双向数据绑定
- Axios 发送异步请求
- LocalStorage 存储Token和用户信息
- CSS3 动画效果

### 1.3 欢迎页面 (index.html)

- 系统欢迎页面
- 环保主题设计
- 跳转登录入口

### 1.4 系统主页 (home.html)

**功能特性**：
- 环保绿色主题，与登录页风格一致
- 顶部导航栏：显示系统名称、用户信息、退出登录
- 功能模块卡片网格布局（3x2）
- 粒子动画背景效果
- 权限控制：系统管理模块仅ADMIN角色可见

**功能模块**：

| 模块名称 | 状态 | 权限要求 | 说明 |
|---------|------|---------|------|
| 环保数据获取 | 可用 | 所有用户 | 实时数据采集、多源数据整合 |
| 系统管理 | 可用 | 仅ADMIN | 用户管理、权限配置 |
| 其他模块 | 待开发 | - | 灰色显示，敬请期待 |

**技术实现**：
- Vue.js 2.x 条件渲染（v-if 控制管理员模块显示）
- CSS Grid 网格布局
- LocalStorage 读取用户角色信息
- 响应式设计，支持移动端适配

### 1.4 API接口调用

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/auth/login | POST | 用户登录 |
| /api/auth/captcha | GET | 获取验证码 |
| /api/auth/userinfo | GET | 获取用户信息 |

## 二、后端工作

### 2.1 项目结构

```
back-end/
├── pom.xml                          # Maven配置
├── sql/
│   └── init.sql                     # 数据库初始化脚本
└── src/main/
    ├── java/com/environment/
    │   ├── EnvironmentHubApplication.java    # Spring Boot启动类
    │   ├── config/
    │   │   ├── CorsFilter.java               # 跨域过滤器
    │   │   ├── JwtAuthFilter.java            # JWT认证过滤器
    │   │   └── WebMvcConfig.java             # Web MVC配置
    │   ├── controller/
    │   │   ├── AuthController.java           # 认证控制器
    │   │   └── SystemController.java         # 系统控制器
    │   ├── mapper/
    │   │   └── UserMapper.java               # 用户Mapper接口
    │   ├── pojo/
    │   │   ├── User.java                     # 用户实体
    │   │   ├── Result.java                   # 统一响应结果
    │   │   ├── LoginRequest.java             # 登录请求参数
    │   │   └── LoginResponse.java            # 登录响应
    │   ├── service/
    │   │   ├── UserService.java              # 用户服务接口
    │   │   └── impl/UserServiceImpl.java     # 用户服务实现
    │   └── util/

    │       └── CaptchaUtil.java              # 验证码工具类
    └── resources/
        ├── application.yml                   # Spring Boot配置
        └── mapper/
            └── UserMapper.xml                # MyBatis映射文件
```

### 2.2 数据库设计

**用户表 (sys_user)**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 用户ID（主键） |
| username | VARCHAR(50) | 用户名（唯一） |
| password | VARCHAR(100) | 密码 |
| real_name | VARCHAR(50) | 真实姓名 |
| email | VARCHAR(100) | 邮箱 |
| phone | VARCHAR(20) | 手机号 |
| department | VARCHAR(100) | 部门 |
| role | VARCHAR(20) | 角色（ADMIN/USER/MONITOR） |
| status | TINYINT | 状态（0-禁用，1-启用） |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| deleted | TINYINT | 删除标志 |

**角色表 (sys_role)**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 角色ID |
| role_name | VARCHAR(50) | 角色名称 |
| role_code | VARCHAR(50) | 角色编码 |
| description | VARCHAR(200) | 角色描述 |
| status | TINYINT | 状态 |

### 2.3 核心功能模块

#### 2.3.1 JWT认证模块

**JwtUtil.java** 功能：
- 生成JWT Token（有效期24小时）
- 解析Token获取用户信息
- 验证Token有效性
- 刷新Token

**JwtAuthFilter.java** 功能：
- 拦截请求验证Token
- 排除登录、验证码等公开接口
- 将用户信息存入Request作用域

#### 2.3.2 验证码模块

**CaptchaUtil.java** 功能：
- 生成4位数字验证码
- 生成指定位数验证码
- 生成数字字母混合验证码
- 验证码比对

#### 2.3.3 用户认证模块

**AuthController.java** 接口：

| 接口路径 | 方法 | 功能 |
|----------|------|------|
| /auth/login | POST | 用户登录 |
| /auth/logout | POST | 用户登出 |
| /auth/captcha | GET | 获取验证码 |
| /auth/userinfo | GET | 获取当前用户信息 |

**SystemController.java** 接口：

| 接口路径 | 方法 | 功能 |
|----------|------|------|
| /system/health | GET | 健康检查 |
| /system/info | GET | 系统信息 |

#### 2.3.4 跨域配置

**CorsFilter.java** 功能：
- 允许所有来源跨域访问
- 支持常用HTTP方法
- 允许携带认证信息

## 三、接口文档

### 3.1 登录接口

**请求**：
```
POST /api/auth/login
Content-Type: application/json

{
    "username": "admin",
    "password": "admin123",
    "captcha": "1234"
}
```

**响应**：
```json
{
    "code": 200,
    "message": "登录成功",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiJ9...",
        "user": {
            "id": 1,
            "username": "admin",
            "realName": "系统管理员",
            "email": "admin@environment.com",
            "department": "信息技术部",
            "role": "ADMIN"
        }
    }
}
```

### 3.2 统一响应格式

```json
{
    "code": 200,
    "message": "提示信息",
    "data": {}
}
```

**状态码说明**：
- 200：成功
- 400：参数错误
- 401：未授权
- 500：服务器错误

## 四、运行说明

### 4.1 数据库初始化

```bash
mysql -u root -p < back-end/sql/init.sql
```

### 4.2 后端启动

```bash
cd back-end
mvn spring-boot:run
```

启动成功后访问：http://localhost:8080/api

### 4.3 前端启动

直接浏览器打开 `front-end/login.html` 或使用 VS Code Live Server 插件。

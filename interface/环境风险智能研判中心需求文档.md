# 智能环境治理中枢平台 - 环境风险智能研判中心需求文档

## 1. 项目背景

当前平台已经具备用户登录、环保数据获取、空气质量因子预测、环保小助手和系统管理等能力。为了让系统从“数据展示与问答工具”升级为“环境治理辅助决策平台”，拟建设 **环境风险智能研判中心**。

该中心将数据获取、模型预测、风险预警、AI 解释和治理建议串联成一条完整业务闭环，帮助监管人员快速判断未来空气质量风险、理解风险来源，并形成可展示、可追溯、可导出的研判结论。

## 2. 建设目标

1. 将空气质量历史数据、未来预测结果和环境知识库进行融合分析。
2. 自动识别 AQI、PM2.5、PM10、O3 等关键因子的风险等级。
3. 为每次研判生成风险原因解释、治理建议和管理简报。
4. 支持预警记录沉淀，形成“发现风险 - 分析原因 - 生成建议 - 跟踪处置”的闭环。
5. 在面试或答辩场景中突出平台的 AI 决策辅助能力，而不是停留在普通 CRUD 系统。

## 3. 用户角色

| 角色 | 使用场景 | 权限说明 |
|------|----------|----------|
| ADMIN | 平台管理、阈值配置、查看全部研判记录 | 拥有全部功能权限 |
| MONITOR | 环境监测、风险研判、预警处置 | 可发起研判、查看和处理预警 |
| USER | 普通查询、查看个人研判结果 | 可发起基础研判，受次数或范围限制 |

## 4. 核心业务流程

1. 用户进入“风险研判中心”，选择城市、日期范围、关注因子和研判模式。
2. 系统读取已抓取的历史空气质量数据；如数据不足，提示是否触发数据抓取。
3. 系统调用预测模型生成未来 7 天空气质量因子预测结果。
4. 风险评分引擎根据预测值、阈值、趋势变化和历史波动计算风险等级。
5. AI 研判助手结合预测结果、环境知识库和规则阈值，生成风险解释与治理建议。
6. 系统保存研判记录，并在达到预警条件时生成预警事件。
7. 用户可查看研判详情、导出研判报告、标记预警处置状态。

## 5. 功能需求

### 5.1 风险研判看板

**功能说明**：展示当前城市或默认城市的环境风险概览。

**核心内容**：

| 模块 | 展示内容 |
|------|----------|
| 风险总览 | 当前最高风险等级、重点污染因子、未来 7 天风险趋势 |
| 城市筛选 | 支持选择宜春、南昌、九江、赣州等已有城市 |
| 指标筛选 | 支持 AQI、PM2.5、PM10、SO2、NO2、CO、O3 |
| 趋势展示 | 展示预测值、风险等级、超标日期和变化趋势 |
| 预警摘要 | 展示待处理、处理中、已处理预警数量 |

**验收标准**：

- 用户进入页面后可以看到默认城市的风险概览。
- 切换城市或因子后，风险总览和趋势数据同步刷新。
- 风险等级需要用低风险、中风险、高风险、严重风险进行区分。

### 5.2 一键智能研判

**功能说明**：用户选择研判条件后，系统自动完成预测、评分、解释和建议生成。

**输入条件**：

| 字段 | 说明 |
|------|------|
| 城市 | 研判城市 |
| 关注因子 | 一个或多个空气质量因子 |
| 预测天数 | 默认 7 天 |
| 研判模式 | 快速研判、标准研判、深度研判 |
| 是否生成报告 | 是否同步生成可导出的研判报告 |

**输出内容**：

| 字段 | 说明 |
|------|------|
| 综合风险等级 | 本次研判的最高风险等级 |
| 重点风险因子 | 贡献最高的污染因子 |
| 风险日期 | 预测可能出现风险的日期 |
| 趋势摘要 | 对未来 7 天趋势的自然语言描述 |
| 成因解释 | AI 生成的风险原因说明 |
| 治理建议 | 面向监管和公众的处置建议 |
| 置信度 | 模型和数据综合置信度 |

**验收标准**：

- 系统能基于已有预测接口完成未来 7 天研判。
- 研判结果必须包含结构化数据和自然语言解释。
- AI 解释需要说明“根据哪些指标或趋势得出结论”。

### 5.3 风险评分与等级判定

**功能说明**：根据预测数据计算风险评分和风险等级。

**评分参考因素**：

| 因素 | 说明 |
|------|------|
| 指标预测值 | AQI、PM2.5、PM10、O3 等预测值 |
| 阈值超限情况 | 是否超过国家空气质量等级参考阈值或系统配置阈值 |
| 趋势变化 | 未来几天是否持续上升 |
| 历史波动 | 与近期历史平均值相比是否异常 |
| 数据完整度 | 历史样本是否充足 |

**风险等级建议**：

| 等级 | 分值范围 | 说明 |
|------|----------|------|
| LOW | 0-39 | 低风险，空气质量整体稳定 |
| MEDIUM | 40-69 | 中风险，存在轻度污染或上升趋势 |
| HIGH | 70-89 | 高风险，可能出现明显污染过程 |
| SEVERE | 90-100 | 严重风险，需要重点关注和处置 |

### 5.4 AI 成因解释与治理建议

**功能说明**：调用环保小助手能力，对结构化预测结果进行解释。

**解释维度**：

- 哪些因子导致风险上升。
- 风险是否具有连续性。
- 是否存在明显异常值或突增趋势。
- 可以采取哪些治理建议。
- 如何向公众发布健康防护建议。

**治理建议分类**：

| 类型 | 示例 |
|------|------|
| 监管建议 | 加强重点区域巡查、关注工业排放和道路扬尘 |
| 企业建议 | 检查废气处理设施运行状态、减少高污染时段作业 |
| 公众建议 | 减少户外运动、佩戴防护口罩、关注空气质量变化 |
| 数据建议 | 补充近 30 天数据、复核异常监测值 |

### 5.5 预警事件管理

**功能说明**：当研判结果达到预警条件时，系统自动生成预警事件。

**预警字段**：

| 字段 | 说明 |
|------|------|
| 预警编号 | 系统自动生成 |
| 城市 | 预警城市 |
| 风险因子 | 触发预警的主要因子 |
| 风险等级 | HIGH 或 SEVERE 时默认生成预警 |
| 触发日期 | 预测出现风险的日期 |
| 预警状态 | 待处理、处理中、已处理、已忽略 |
| 处置备注 | 监测员或管理员填写 |

**验收标准**：

- 高风险和严重风险研判可自动生成预警。
- 用户可以查看预警列表和详情。
- MONITOR 和 ADMIN 可更新预警处理状态。

### 5.6 研判报告生成

**功能说明**：根据一次研判结果生成可导出的报告内容。

**报告结构**：

1. 基本信息：城市、时间、研判人、生成时间。
2. 数据来源：历史数据范围、预测模型、知识库来源。
3. 风险摘要：综合等级、重点因子、风险日期。
4. 趋势分析：未来 7 天预测表格和变化说明。
5. AI 研判：成因解释、治理建议、公众提示。
6. 处置建议：监管动作和后续跟踪建议。

**导出形式**：

- 第一阶段支持 Markdown 或 HTML 文本。
- 后续可扩展为 PDF、Word 或系统内报告页面。

## 6. 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 标准研判接口建议 30 秒内返回；深度研判可放宽到 120 秒 |
| 安全 | 所有研判接口均需登录认证，阈值配置仅 ADMIN 可修改 |
| 可追溯 | 每次研判需保存输入参数、预测结果、AI 解释和生成时间 |
| 可降级 | AI 服务异常时，系统仍需返回结构化风险评分和基础建议 |
| 可扩展 | 后续可增加水质、噪声、企业排放等风险类型 |

| 区域 | 内容 |

## 9. 分阶段实现建议

### 第一阶段：答辩亮点闭环

1. 新增风险研判中心页面入口。
2. 复用现有预测接口生成未来 7 天预测。
3. 增加风险评分、风险等级和 AI 解释接口。
4. 展示研判结果并支持保存记录。

### 第二阶段：预警管理

1. 保存高风险和严重风险预警。
2. 增加预警列表、详情和状态更新。
3. 支持管理员配置阈值。

### 第三阶段：报告与扩展

1. 生成 Markdown/HTML 研判报告。
2. 支持导出报告。
3. 扩展水质、噪声、企业排放等风险类型。

## 10. 面试展示话术建议

本项目从原来的环境数据管理平台升级为环境风险智能研判中心。系统不只是展示数据，而是通过数据采集、趋势预测、风险评分、AI 成因解释和治理建议，形成环境治理辅助决策闭环。这样可以体现数据工程、模型服务、智能体应用和业务系统设计的综合能力。

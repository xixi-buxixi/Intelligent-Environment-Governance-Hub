## 通用说明

### 请求头

除公开接口外，所有接口需在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 统一响应格式

```json
{
    "code": 200,
    "message": "提示信息",
    "data": {}
}
```

### 状态码说明

| 状态码 | 说明 |
|-------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未授权/Token失效 |
| 403 | 权限不足 |
| 500 | 服务器错误 |

## 1. 认证模块

### 1.1 用户登录

**接口地址**：`POST /api/auth/login`

**是否鉴权**：否

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |
| captcha | String | 是 | 验证码 |

**请求示例**：

```json
{
    "username": "admin",
    "password": "admin123",
    "captcha": "1234"
}
```

**响应参数**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 返回数据 |
| data.token | String | JWT Token |
| data.user | Object | 用户信息 |
| data.user.id | Long | 用户ID |
| data.user.username | String | 用户名 |
| data.user.realName | String | 真实姓名 |
| data.user.email | String | 邮箱 |
| data.user.phone | String | 手机号 |
| data.user.department | String | 部门 |
| data.user.role | String | 角色（ADMIN/USER/MONITOR） |

**响应示例**：

```json
{
    "code": 200,
    "message": "登录成功",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiJ9...",
        "user": {
            "id": 1,
            "username": "admin",
            "realName": "系统管理员",
            "email": "admin@example.com",
            "phone": "13800138000",
            "department": "信息技术部",
            "role": "ADMIN"
        }
    }
}
```

## 2. 用户管理模块

> 以下接口均需要 ADMIN 角色权限

### 2.1 获取用户列表

**接口地址**：`GET /api/system/user/list`

**是否鉴权**：是（需 ADMIN 角色）

**请求参数**（Query String）：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| page | Integer | 否 | 当前页码，默认 1 |
| size | Integer | 否 | 每页条数，默认 10 |

**请求示例**：

```
GET /api/system/user/list?page=1&size=10
```

**响应参数**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 返回数据 |
| data.records | Array | 用户列表（或使用 list） |
| data.total | Long | 总记录数 |
| data.current | Integer | 当前页码 |
| data.size | Integer | 每页条数 |
| data.pages | Integer | 总页数 |

**records 数组元素结构**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| id | Long | 用户ID |
| username | String | 用户名 |
| realName | String | 真实姓名 |
| email | String | 邮箱 |
| phone | String | 手机号 |
| department | String | 部门 |
| role | String | 角色（ADMIN/USER/MONITOR） |
| status | Integer | 状态（0-禁用，1-启用） |
| createTime | String | 创建时间 |
| updateTime | String | 更新时间 |

**响应示例**：

```json
{
    "code": 200,
    "message": "成功",
    "data": {
        "records": [
            {
                "id": 1,
                "username": "admin",
                "realName": "系统管理员",
                "email": "admin@example.com",
                "phone": "13800138000",
                "department": "信息技术部",
                "role": "ADMIN",
                "status": 1,
                "createTime": "2026-03-10 10:00:00",
                "updateTime": "2026-03-10 10:00:00"
            },
            {
                "id": 2,
                "username": "user01",
                "realName": "张三",
                "email": "zhangsan@example.com",
                "phone": "13800138001",
                "department": "运维部",
                "role": "USER",
                "status": 1,
                "createTime": "2026-03-10 11:00:00",
                "updateTime": "2026-03-10 11:00:00"
            }
        ],
        "total": 50,
        "current": 1,
        "size": 10,
        "pages": 5
    }
}
```

### 2.6 修改用户权限

**接口地址**：`POST /api/system/user/updateRole`

**是否鉴权**：是（需 ADMIN 角色）

**接口说明**：管理员修改用户的角色权限，需要输入管理员密码进行确认。

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| userId | Long | 是 | 要修改的用户ID |
| newRole | String | 是 | 新角色（ADMIN/USER/MONITOR） |
| password | String | 是 | 当前管理员的登录密码（用于身份确认） |

**请求示例**：

```json
{
    "userId": 2,
    "newRole": "MONITOR",
    "password": "admin123"
}
```

**响应参数**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 返回数据（null） |

**响应示例**：

成功：
```json
{
    "code": 200,
    "message": "权限修改成功",
    "data": null
}
```

密码错误：
```json
{
    "code": 400,
    "message": "密码错误",
    "data": null
}
```

不能修改自己的角色：
```json
{
    "code": 403,
    "message": "不能修改自己的角色",
    "data": null
}
```

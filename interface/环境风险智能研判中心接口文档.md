## 1. 通用说明

### 1.1 请求前缀

建议统一使用以下接口前缀：

```
/api/risk
```

### 1.2 请求头

除特别说明外，所有接口均需携带 JWT Token：

```
Authorization: Bearer <token>
Content-Type: application/json
```

### 1.3 统一响应格式

```json
{
    "code": 200,
    "message": "成功",
    "data": {}
}
```

### 1.4 状态码说明

| 状态码 | 说明 |
|-------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录或 Token 失效 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### 1.5 枚举说明

**风险等级 riskLevel**

| 值 | 说明 |
|----|------|
| LOW | 低风险 |
| MEDIUM | 中风险 |
| HIGH | 高风险 |
| SEVERE | 严重风险 |

**研判模式 analysisMode**

| 值 | 说明 |
|----|------|
| FAST | 快速研判，仅返回评分和基础建议 |
| STANDARD | 标准研判，返回预测、评分和 AI 解释 |
| DEEP | 深度研判，返回完整解释、建议和报告 |

**预警状态 alertStatus**

| 值 | 说明 |
|----|------|
| PENDING | 待处理 |
| PROCESSING | 处理中 |
| RESOLVED | 已处理 |
| IGNORED | 已忽略 |

## 2. 风险研判看板

### 2.1 获取风险看板概览

**接口地址**：`GET /api/risk/overview`

**是否鉴权**：是

**接口说明**：获取指定城市的风险概览、未来趋势和预警摘要，供风险研判中心首页展示。

**请求参数**（Query String）：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| city | String | 否 | 城市，默认宜春 |
| factors | String | 否 | 因子列表，英文逗号分隔，如 AQI,PM25,O3 |
| horizonDays | Integer | 否 | 预测天数，默认 7 |

**请求示例**：

```
GET /api/risk/overview?city=宜春&factors=AQI,PM25,O3&horizonDays=7
```

**响应参数**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| city | String | 城市 |
| highestRiskLevel | String | 最高风险等级 |
| riskScore | Integer | 综合风险评分 |
| keyFactor | String | 重点风险因子 |
| confidence | Double | 置信度，0-1 |
| trendSummary | String | 趋势摘要 |
| pendingAlertCount | Integer | 待处理预警数 |
| processingAlertCount | Integer | 处理中预警数 |
| resolvedAlertCount | Integer | 已处理预警数 |
| trendItems | Array | 趋势明细 |

**trendItems 数组元素结构**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| date | String | 日期，YYYY-MM-DD |
| factor | String | 因子 |
| predictedValue | Double | 预测值 |
| riskLevel | String | 风险等级 |
| riskScore | Integer | 风险评分 |
| exceedThreshold | Boolean | 是否超过阈值 |

**响应示例**：

```json
{
    "code": 200,
    "message": "成功",
    "data": {
        "city": "宜春",
        "highestRiskLevel": "HIGH",
        "riskScore": 78,
        "keyFactor": "PM25",
        "confidence": 0.82,
        "trendSummary": "未来 7 天 PM2.5 存在持续上升趋势，5 月 9 日附近可能达到高风险水平。",
        "pendingAlertCount": 2,
        "processingAlertCount": 1,
        "resolvedAlertCount": 8,
        "trendItems": [
            {
                "date": "2026-05-07",
                "factor": "PM25",
                "predictedValue": 62.5,
                "riskLevel": "MEDIUM",
                "riskScore": 58,
                "exceedThreshold": false
            },
            {
                "date": "2026-05-09",
                "factor": "PM25",
                "predictedValue": 84.2,
                "riskLevel": "HIGH",
                "riskScore": 78,
                "exceedThreshold": true
            }
        ]
    }
}
```

## 3. 智能研判

### 3.1 发起智能研判

**接口地址**：`POST /api/risk/analyze`

**是否鉴权**：是

**接口说明**：根据城市、因子和研判模式生成完整风险研判结果。该接口内部可复用现有 `/api/predict/air-quality` 预测能力，并结合规则评分和 AI 解释生成研判结论。

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| city | String | 是 | 城市 |
| factors | Array | 是 | 因子列表 |
| horizonDays | Integer | 否 | 预测天数，默认 7 |
| analysisMode | String | 否 | FAST/STANDARD/DEEP，默认 STANDARD |
| generateReport | Boolean | 否 | 是否同步生成报告，默认 false |
| saveRecord | Boolean | 否 | 是否保存研判记录，默认 true |

**请求示例**：

```json
{
    "city": "宜春",
    "factors": ["AQI", "PM25", "PM10", "O3"],
    "horizonDays": 7,
    "analysisMode": "STANDARD",
    "generateReport": true,
    "saveRecord": true
}
```

**响应参数**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| analysisId | Long | 研判记录 ID，未保存时为空 |
| city | String | 城市 |
| factors | Array | 因子列表 |
| horizonDays | Integer | 预测天数 |
| riskLevel | String | 综合风险等级 |
| riskScore | Integer | 综合风险评分 |
| keyFactor | String | 重点风险因子 |
| riskDates | Array | 风险日期 |
| confidence | Double | 置信度 |
| trendSummary | String | 趋势摘要 |
| causeExplanation | String | AI 成因解释 |
| governanceSuggestions | Array | 治理建议 |
| publicAdvice | String | 公众健康提示 |
| predictions | Array | 预测明细 |
| alertCreated | Boolean | 是否生成预警 |
| alertId | Long | 预警 ID |
| reportContent | String | 报告内容，generateReport=true 时返回 |

**predictions 数组元素结构**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| date | String | 日期 |
| values | Object | 各因子预测值 |
| dailyRiskLevel | String | 当日风险等级 |
| dailyRiskScore | Integer | 当日风险评分 |
| mainFactor | String | 当日主要风险因子 |

**响应示例**：

```json
{
    "code": 200,
    "message": "研判完成",
    "data": {
        "analysisId": 1001,
        "city": "宜春",
        "factors": ["AQI", "PM25", "PM10", "O3"],
        "horizonDays": 7,
        "riskLevel": "HIGH",
        "riskScore": 78,
        "keyFactor": "PM25",
        "riskDates": ["2026-05-09", "2026-05-10"],
        "confidence": 0.82,
        "trendSummary": "未来 7 天 AQI 整体呈上升趋势，PM2.5 是主要风险因子。",
        "causeExplanation": "预测结果显示 PM2.5 连续两天处于较高水平，且 AQI 同步升高，说明颗粒物污染可能是本轮风险的主要来源。",
        "governanceSuggestions": [
            "建议加强重点道路扬尘管控和施工工地巡查。",
            "建议关注工业企业废气治理设施运行状态。",
            "建议提前发布空气质量提醒。"
        ],
        "publicAdvice": "敏感人群应减少长时间户外活动，外出时可佩戴防护口罩。",
        "predictions": [
            {
                "date": "2026-05-09",
                "values": {
                    "AQI": 116,
                    "PM25": 84.2,
                    "PM10": 132.5,
                    "O3": 148.0
                },
                "dailyRiskLevel": "HIGH",
                "dailyRiskScore": 78,
                "mainFactor": "PM25"
            }
        ],
        "alertCreated": true,
        "alertId": 501,
        "reportContent": "# 宜春空气质量风险研判报告\n\n..."
    }
}
```

### 3.2 重新生成 AI 解释

**接口地址**：`POST /api/risk/analyze/{analysisId}/explanation`

**是否鉴权**：是

**接口说明**：当用户修改研判模式或 AI 解释生成失败时，可基于已有研判记录重新生成解释和建议。

**路径参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| analysisId | Long | 是 | 研判记录 ID |

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| analysisMode | String | 否 | STANDARD/DEEP，默认 STANDARD |
| focus | String | 否 | 解释侧重点，如监管建议、公众提示、成因分析 |

**请求示例**：

```json
{
    "analysisMode": "DEEP",
    "focus": "监管建议"
}
```

**响应示例**：

```json
{
    "code": 200,
    "message": "解释生成成功",
    "data": {
        "analysisId": 1001,
        "causeExplanation": "PM2.5 和 PM10 同时升高，说明颗粒物污染贡献较高。",
        "governanceSuggestions": [
            "加强扬尘源巡查。",
            "关注重点排放企业在线监测数据。"
        ],
        "publicAdvice": "敏感人群建议减少户外活动。"
    }
}
```

## 4. 研判记录

### 4.1 获取研判记录列表

**接口地址**：`GET /api/risk/records`

**是否鉴权**：是

**接口说明**：分页查询研判历史。普通用户默认只能查看自己的记录，ADMIN 和 MONITOR 可查看全部记录。

**请求参数**（Query String）：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| page | Integer | 否 | 页码，默认 1 |
| size | Integer | 否 | 每页条数，默认 10 |
| city | String | 否 | 城市 |
| riskLevel | String | 否 | 风险等级 |
| startDate | String | 否 | 开始日期，YYYY-MM-DD |
| endDate | String | 否 | 结束日期，YYYY-MM-DD |

**响应参数**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| records | Array | 研判记录列表 |
| total | Long | 总数 |
| current | Integer | 当前页 |
| size | Integer | 每页条数 |
| pages | Integer | 总页数 |

**records 数组元素结构**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| id | Long | 研判记录 ID |
| city | String | 城市 |
| factors | Array | 因子列表 |
| riskLevel | String | 风险等级 |
| riskScore | Integer | 风险评分 |
| keyFactor | String | 重点因子 |
| confidence | Double | 置信度 |
| createUserName | String | 发起人 |
| createTime | String | 创建时间 |

**响应示例**：

```json
{
    "code": 200,
    "message": "成功",
    "data": {
        "records": [
            {
                "id": 1001,
                "city": "宜春",
                "factors": ["AQI", "PM25"],
                "riskLevel": "HIGH",
                "riskScore": 78,
                "keyFactor": "PM25",
                "confidence": 0.82,
                "createUserName": "监测员",
                "createTime": "2026-05-06 16:30:00"
            }
        ],
        "total": 1,
        "current": 1,
        "size": 10,
        "pages": 1

}
```

## 7. 报告生成

### 7.1 生成研判报告

**接口地址**：`POST /api/risk/reports`

**是否鉴权**：是

**接口说明**：根据已有研判记录生成报告内容。若研判时已生成报告，可直接返回已有报告。

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| analysisId | Long | 是 | 研判记录 ID |
| format | String | 否 | MARKDOWN/HTML，默认 MARKDOWN |
| regenerate | Boolean | 否 | 是否重新生成，默认 false |

**请求示例**：

```json
{
    "analysisId": 1001,
    "format": "MARKDOWN",
    "regenerate": false
}
```

**响应参数**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| analysisId | Long | 研判记录 ID |
| reportTitle | String | 报告标题 |
| format | String | 报告格式 |
| content | String | 报告正文 |
| generatedAt | String | 生成时间 |

**响应示例**：

```json
{
    "code": 200,
    "message": "报告生成成功",
    "data": {
        "analysisId": 1001,
        "reportTitle": "宜春空气质量风险研判报告",
        "format": "MARKDOWN",
        "content": "# 宜春空气质量风险研判报告\n\n## 一、风险摘要\n\n综合风险等级：高风险。",
        "generatedAt": "2026-05-06 17:30:00"
    }
}
```

### 7.2 导出研判报告

**接口地址**：`GET /api/risk/reports/{analysisId}/export`

**是否鉴权**：是

**接口说明**：导出研判报告文件。第一阶段建议导出 Markdown 或 HTML，后续可扩展 PDF、Word。

**请求参数**（Query String）：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| format | String | 否 | MARKDOWN/HTML，默认 MARKDOWN |

**请求示例**：

```
GET /api/risk/reports/1001/export?format=MARKDOWN
```

**响应说明**：

文件流响应。

**响应头示例**：

```
Content-Type: text/markdown;charset=UTF-8
Content-Disposition: attachment; filename=risk-report-1001.md
```

## 8. 与现有接口的关系

风险研判中心优先复用现有能力，避免重复建设：

| 现有能力 | 已有接口 | 复用方式 |
|----------|----------|----------|
| 用户认证 | `/api/auth/*` | 继续使用 JWT 鉴权 |

## 通用说明

### 请求头

所有接口需在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 统一响应格式

```json
{
    "code": 200,
    "message": "提示信息",
    "data": {}
}
```

### 状态码说明

| 状态码 | 说明 |
|-------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未授权/Token失效 |
| 403 | 权限不足/获取次数已用完 |
| 500 | 服务器错误 |

### 角色权限说明

| 角色 | 获取次数限制 | 说明 |
|-----|------------|------|
| ADMIN | 无限制 | 管理员可无限次获取数据 |
| USER | 3次/天 | 普通用户每日限制3次 |
| MONITOR | 无限制 | 监测员可无限次获取数据 |

## 1. 数据获取初始化

### 1.1 获取初始化信息

**接口地址**：`GET /api/data/init-info`

**是否鉴权**：是

**接口说明**：获取数据源列表、城市列表、当前用户的获取次数限制等信息。页面加载时调用。

**请求参数**：无

**请求示例**：

```
GET /api/data/init-info
Header: Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

**响应参数**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Object | 返回数据 |
| data.sources | Array | 数据源列表 |
| data.cities | Array | 城市列表 |
| data.fetchLimit | Object | 获取次数限制信息 |

**sources 数组元素结构**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| id | Long | 数据源ID |
| sourceName | String | 数据源名称 |
| description | String | 数据源描述 |
| sourceUrl | String | 数据源地址 |
| dataTypes | String | 支持的数据类型（JSON数组字符串） |

**fetchLimit 对象结构**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| hasLimit | Boolean | 是否有限制（true-有限制，false-无限制） |
| remainingCount | Integer | 今日剩余次数 |
| dailyLimit | Integer | 每日总限制次数 |

> **注意**：前端会根据用户角色自行判断是否显示限制信息。对于 ADMIN 用户，前端会忽略后端返回的 fetchLimit，直接显示无限制。

**响应示例**：

```json
{
    "code": 200,
    "message": "成功",
    "data": {
        "sources": [
            {
                "id": 1,
                "sourceName": "中国环境监测总站",
                "description": "国家级环境监测数据平台，提供空气质量、水质等环境监测数据",
                "sourceUrl": "https://www.cnemc.cn",
                "dataTypes": "[\"AQI\", \"PM2.5\", \"PM10\", \"SO2\", \"NO2\", \"CO\", \"O3\"]"
            },
            {
                "id": 2,
                "sourceName": "生态环境部数据中心",
                "description": "生态环境部官方数据发布平台",
                "sourceUrl": "https://data.mee.gov.cn",
                "dataTypes": "[\"空气质量\", \"水质数据\", \"噪声数据\"]"
            }
        ],
        "cities": [
            "北京", "上海", "广州", "深圳", "杭州",
            "南京", "武汉", "成都", "西安", "重庆",
            "天津", "苏州", "郑州", "长沙", "青岛"
        ],
        "fetchLimit": {
            "hasLimit": true,
            "remainingCount": 2,
            "dailyLimit": 3
        }
    }
}
```

**管理员响应示例**（后端返回示例，前端会忽略）：

```json
{
    "code": 200,
    "message": "成功",
    "data": {
        "sources": [...],
        "cities": [...],
        "fetchLimit": {
            "hasLimit": false,
            "remainingCount": 999,
            "dailyLimit": 999
        }
    }
}
```

## 2. 获取历史记录

### 2.1 获取数据获取历史记录

**接口地址**：`GET /api/data/records`

**是否鉴权**：是

**接口说明**：获取当前用户的数据获取历史记录，按时间倒序排列。

**请求参数**（Query String）：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| page | Integer | 否 | 当前页码，默认 1 |
| size | Integer | 否 | 每页条数，默认 20 |

**请求示例**：

```
GET /api/data/records?page=1&size=20
Header: Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

**响应参数**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | Array | 历史记录列表 |

**data 数组元素结构**：

| 参数名 | 类型 | 说明 |
|-------|------|------|
| id | Long | 记录ID |
| userId | Long | 用户ID |
| sourceId | Long | 数据源ID |
| sourceName | String | 数据源名称 |
| city | String | 城市 |
| startDate | String | 起始日期（格式：YYYY-MM-DD） |
| endDate | String | 终止日期（格式：YYYY-MM-DD） |
| dataType | String | 数据类型 |
| recordCount | Integer | 获取的记录数 |
| createTime | String | 获取时间（格式：YYYY-MM-DD HH:mm:ss） |

**响应示例**：

## 3. 数据获取与导出

### 3.1 获取数据并导出CSV

**接口地址**：`POST /api/data/fetch`

**是否鉴权**：是

**接口说明**：根据查询条件获取环境数据，并以CSV文件格式返回。普通用户每次调用会消耗一次获取次数。

**请求头**：

```
Authorization: Bearer <token>
Content-Type: application/json
```

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| sourceId | Long | 是 | 数据源ID |
| city | String | 是 | 城市 |
| startDate | String | 是 | 起始日期（格式：YYYY-MM-DD） |
| endDate | String | 是 | 终止日期（格式：YYYY-MM-DD） |
| dataType | String | 是 | 数据类型 |

**请求示例**：

```json
{
    "sourceId": 1,
    "city": "北京",
    "startDate": "2026-03-01",
    "endDate": "2026-03-15",
    "dataType": "AQI"
}
```

**响应说明**：

成功时返回 CSV 文件流，响应头包含：

| 响应头 | 值 | 说明 |
|-------|-----|------|
| Content-Type | text/csv;charset=utf-8 | CSV格式 |
| Content-Disposition | attachment; filename="xxx.csv" | 文件名 |

**CSV 文件格式**：

文件名格式：`{城市}_{数据类型}_{起始日期}_{终止日期}.csv`

示例文件内容：

```csv
日期,城市,AQI,PM2.5,PM10,SO2,NO2,CO,O3,空气质量
2026-03-01,北京,85,62,98,12,38,0.9,45,良
2026-03-02,北京,92,68,105,15,42,1.1,52,良
2026-03-03,北京,78,55,88,10,35,0.8,40,良
...
```

**错误响应示例**：

次数已用完：

```json
{
    "code": 403,
    "message": "今日获取次数已用完，请明天再试",
    "data": null
}
```

参数错误：

```json
{
    "code": 400,
    "message": "参数错误：起始日期不能晚于终止日期",
    "data": null
}
```

数据源不存在：

```json
{
    "code": 400,
    "message": "数据源不存在",
    "data": null
}
```

无数据：

```json
{
    "code": 200,
    "message": "未查询到符合条件的数据",
    "data": null
}
```

## 4. 数据类型说明

### 4.1 空气质量相关数据类型

| 数据类型 | 说明 | 单位 |
|---------|------|------|
| AQI | 空气质量指数 | 无量纲 |
| PM2.5 | 细颗粒物浓度 | μg/m³ |
| PM10 | 可吸入颗粒物浓度 | μg/m³ |
| SO2 | 二氧化硫浓度 | μg/m³ |
| NO2 | 二氧化氮浓度 | μg/m³ |
| CO | 一氧化碳浓度 | mg/m³ |
| O3 | 臭氧浓度 | μg/m³ |

### 4.2 水质相关数据类型

| 数据类型 | 说明 | 单位 |
|---------|------|------|
| pH | 酸碱度 | 无量纲 |
| DO | 溶解氧 | mg/L |
| COD | 化学需氧量 | mg/L |
| NH3N | 氨氮 | mg/L |
| TP | 总磷 | mg/L |
| TN | 总氮 | mg/L |

## 5. 业务流程说明

### 5.1 页面初始化流程

```
1. 页面加载 → checkLogin()
   ├── 检查 token 和 userInfo 是否存在
   ├── 解析用户信息，获取 userName 和 userRole
   └── 如果是 ADMIN，直接设置 fetchLimit.hasLimit = false

2. 调用 initData()
   ├── 请求 GET /api/data/init-info
   ├── 获取数据源列表 sources
   ├── 获取城市列表 cities
   └── 如果是普通用户，使用后端返回的 fetchLimit

3. 调用 loadHistory()
   └── 请求 GET /api/data/records 获取历史记录
```

### 5.2 数据获取流程

```
1. 用户填写表单（数据源、城市、日期范围、数据类型）

2. 点击"导出为CSV"按钮
   ├── 前端验证表单完整性
   ├── 前端验证获取次数（普通用户剩余次数 > 0）
   └── 发送 POST /api/data/fetch 请求

3. 后端处理
   ├── 验证 Token
   ├── 验证参数
   ├── 检查用户权限和获取次数（普通用户）
   ├── 查询数据
   ├── 生成 CSV 文件
   ├── 记录获取历史
   └── 返回文件流

4. 前端处理响应
   ├── 创建 Blob 对象
   ├── 触发浏览器下载
   └── 刷新页面数据（initData）
```

### 5.3 前端角色判断逻辑

```javascript
// 在 checkLogin() 中
if (userRole.value === 'ADMIN') {
    fetchLimit.value = {
        hasLimit: false,
        remainingCount: 999,
        dailyLimit: 999
    };
}

// 在 initData() 中
if (userRole.value !== 'ADMIN' && data.fetchLimit) {
    fetchLimit.value = data.fetchLimit;
}
```
